## Observations
- Most Marketplace functionality already exists inside `apps/marketplace-app`; replacing it with Senderrplace is primarily a rebrand + configuration/messaging effort coupled with Senderrplace-specific domain contracts (booking links, merchant directory) in Firestore functions.
- Senderrplace introduces booking links and merchant fingerprinting logic; ensure UI surfaces those features (e.g., via a dedicated onboarding flow or button in the seller dashboard).
- To minimize risk we can keep the current routing/feature structure and progressively swap branding strings + hosting target while using the existing Firebase functions for payments/orders.

## Next Steps
